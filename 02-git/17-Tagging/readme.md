# Tagging

У Git теги (tags) використовуються для позначення важливих точок в історії репозиторію, наприклад, релізів чи майлстоунів.

- **Позначення релізів:** Теги часто використовують для маркування версій проекту (наприклад, v1.0, v2.0).
- **Типи тегів:** Існують lightweight (простий тег, лише посилання на коміт) та annotated (анотований тег з додатковою інформацією: автор, дата, повідомлення).

---

## Managing Tags

Тег у Git — це іменоване посилання на конкретний коміт в історії проекту.

- **Створення тегу:**
  - Простий тег:

    ```sh
    git tag <name> [commit-hash]
    ```

  - Анотований тег:

    ```sh
    git tag -a <name> -m "повідомлення" [commit-hash]
    ```

- **Перегляд тегів:**

  ```sh
  git tag
  ```

- **Видалення тегу:**

  ```sh
  git tag -d <tag-name>
  ```

Теги використовують для позначення релізів, майлстоунів або інших важливих подій у проекті.

---

## Pushing Tags

Щоб поділитися локальними тегами з віддаленим репозиторієм:

- **Відправити один тег:**

  ```sh
  git push origin <tag-name>
  ```

- **Відправити всі теги:**

  ```sh
  git push --tags
  ```

---

## Checkout Tags

Щоб перейти до стану репозиторію на момент створення тегу:

```sh
git checkout <tag-name>
```

Це дозволяє переглянути або протестувати код у певній версії проекту.

---

## GitHub Releases

GitHub Releases — це функція для створення, пакування та розповсюдження версій програмного забезпечення.
Вона дозволяє створювати релізи на основі тегів, додавати release notes, прикріплювати бінарні файли (наприклад, збірки чи архіви), і зручно повідомляти спільноту про зміни.
Релізи інтегруються з тегами Git і можуть бути частиною CI/CD-процесу.

### Interview Questions

- Для чого використовуються теги у Git?
- Яка різниця між lightweight і annotated тегами?
- Як створити, переглянути, видалити та відправити тег у Git?
- Як працює GitHub Releases і чим він корисний?

### What to Answer

- Теги використовують для позначення важливих точок (релізів) в історії проекту.
- Lightweight тег — це просте посилання на коміт, annotated — містить додаткову інформацію (автор, дата, повідомлення).
- Створити: `git tag <name>` або `git tag -a <name> -m "msg"`, переглянути: `git tag`, видалити: `git tag -d <name>`, відправити: `git push origin <name>`.
- GitHub Releases дозволяє створювати релізи на основі тегів, додавати release notes і файли, що спрощує розповсюдження та комунікацію з користувачами.
